# Public Roadmap

## Guiding rule

Public promises must stay narrower than real evidence. We ship what we can
verify on real hosts, then widen the surface area one lane at a time.

## Available now

- public repo with `LICENSE`, contribution, security, privacy, and reward docs
- TestFlight app entry
- dataset portal entry
- hardware guidance for `Lite`, `Stereo`, and `Pro`
- agent guides and reusable prompts for Codex, Claude, and OpenClaw
- public `doctor`, `readiness`, `install`, and `basic-e2e` flows
- verified `basic` live evidence on:
  - dedicated `Linux x86_64`
  - local `macOS arm64`

## Next up

- wait for GitHub Dependabot to refresh and clear the patched lockfile alerts
- complete Windows public live evidence
- complete Jetson public live evidence
- complete `Stereo` true-hardware acceptance
- complete `Pro` true-hardware acceptance

## After that

- expand public download validation beyond the current `basic` bundle subset
- improve dataset packaging examples and contributor-facing validation guidance
- make reward reporting and dataset contribution feedback more transparent
- add more screenshots and walkthrough artifacts for first-time contributors

## Not on the public promise line until verified

- claiming full multi-host one-command parity
- claiming full `Stereo / Pro` automation
- claiming Windows or Jetson support as complete without fresh live evidence
